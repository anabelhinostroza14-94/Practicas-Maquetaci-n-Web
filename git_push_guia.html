Guía rápida para hacer git push

Versión HTML generada a partir del archivo PHP original.

0) Comprobar qué remoto está configurado:

git remote -v
git remote add origin https://github.com/usuario/repositorio.git

1) Verifica tu rama actual:

git branch

2) Agregar y confirmar los cambios:

git add .
git commit -m "Mensaje del commit"

3) Hacer push inicial:

git push -u origin master

4) Si tu rama es 'main':

git push -u origin main

SIGNIFICADO

-u : Vincula la rama local con la remota para futuros 'push' y 'pull'.

origin : Nombre del remoto (por defecto apunta a tu repo en GitHub).

master/main : Nombre de la rama que estás subiendo.

ERRORES FRECUENTES

1) error: src refspec main does not match any

• Estás intentando subir 'main' pero tu rama local se llama distinto.

• Solución: usa 'git branch' para ver el nombre correcto.

2) rejected (non-fast-forward)

• Tu remoto tiene cambios que tu local no tiene.

• Solución: primero trae los cambios con:

git pull --rebase origin <rama>

luego haz 'git push'.

3) fatal: No commits yet

• Aún no hiciste un commit.

• Solución:

git add .
git commit -m "Primer commit"
git push -u origin master
